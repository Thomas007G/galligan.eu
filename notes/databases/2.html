Edgar "Ted" Codd introduced the concept of relational databases concept in 1970

Relational Databases are based on tables => Conceptually intuitive, simple and elegant.

A DB consists of a set of named relations (aka tables) Relation:

  • A set of tuples (aka rows) with identical column structure
  • Each column named with an attribute (must be distinct)
  • Values in each column of uniform "type" drawn from some set (domain)

Values stored must be atomic (elementary and unstructured) ie. ints, floats, bools etc. Cant have lists, sets, arrays as single values

Database Schema: Structure of the database, how it is organised.

Database Instance: Any values populating the DB conforming to the schema

relation comes from how there is data with > 1 dataset ie. binary

What is SQL

  • Language for expressing management, manipulation and querying of relational databases

Support:

  • Good: "Standard" DB language; most commonly supported => free open-source
  • Bad: Most DBMs adopt own dialect => software generally isn't transferrable

Several aspects:

Data Manipulation Language(DML)

  • Notation for expressing
  • management of DB
  • manipulations: adding/removing/updating tuples
  • queries to interrogate DB Data Definition Language(DDL)

Query results are in a table, including specified columns and rows only, leaving the original table completely untouched
