Two-Table Queries

Redundancy

  • Replicated data --> Must go to great lengths to avoid, as makes getting or changing data on multiple rows is inefficient.

Cartesion products

SELECT * FROM persons CROSS JOIN favourite_foods

rows = rows(persons) X rows(favourite_foods)

cols = cols(persons) X cols(favourite_foods);

Every Singe Combination --> Often, cross-joins are meaningless

However...

SELECT * FROM persons JOIN favourite_foods ON /*or WHERE */ persons.person_id = favourite_foods.person_id;

Removes everything but combined rows where the person_id match
