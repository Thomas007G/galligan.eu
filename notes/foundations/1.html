An Algorithm is a set of instructions which state how a task is to be performed

Al-Khwarizmi --> Origin of term "Algorithm"

  • An algorithm is an ordered, deterministic, terminating set of instructions

Ordered

  • Must be in correct order, otherwise algorithm does not make sense
  • An example is long division
    • Divide
    • Subtract
    • Carry Down
    • When no more numbers to carry down, the answer is the number above the line
  • The algorithm wouldn't work if the order was switched

Deterministic

  • Instructions must be clear, unambiguous and deterministic forwhoever or whatever is going to interpret them
    • A computer cannot compute "common-sense"

"There is a man knocked down every 2 minutes" --> ambiguous, as it could relate to any man in the world, or a specific man, who gets knocked down constantly.

Executable

  • Are the instructions possible to carry out?
Float up the steps /* impossible, as a human cannot carry out this instruction */

Terminating

  • An algorithm must have an end... but this isn't always the case... so the definition is obsolete(?)

Programming is mostly about algorithms, so you're better off getting used to them sooner rather than later.
