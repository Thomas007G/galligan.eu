Overview of Programming Fundamentals

Everything can roughly be described as being a function or a variable.

print("String Goes Here")
  • print is a function => performs something
name = "John" print(name)
  • name is a variable => holds data
  • Functions perform tasks
  • Variables store values

Type variances

  • intis an integer, ie. a number, not a piece of text

  • str is a piece of text, ie. a name of something

    • An int must be converted to a str to be read as text, and a str must be converted to an int using libraries or dictionaries to be read as an integer

    • Luckily, Python is a dynamically-typed language, ie. types of a variable can change. Statically-typed languages must have data exported to another variable to change the type of data

How a computer runs code

A computer processor only understands binary (1s and 0s). All the processor does is compare these binary digits called bits. There are 8 bits in a byte.

There are two ways a computer understands code. It can be interpreted, like python, or java. Or it can be compiled like C or C++

  • Interpreted implies that an interpreter interprets the code, line-by-line and is converted to bytecode
  • Compiled languages are converted to binaries or machine code
