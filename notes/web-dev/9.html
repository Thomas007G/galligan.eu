Responsive Web Design (RWD)

  • Flexible design (liquid) layout, based on a css grid
  • Flexible images & media
  • Media queries
  • Least-capable client first (DFA)

Images

  • Breakpoints of image pixel sizes used be the standard, but now, there is a wide range of viewport sizes, so this is obsolete
  • Now, we use media queries to change image size in a liquid manner so that we keep the image so as not to look bad at any viewport size
  • Should keep characters per line between at most 45-75, as it becomes difficult to read any more than this. Hence use media queries for 50em
  • Layout viewport may be larger than visual viewport, ex iPhone

Fix:

<meta name="viewport" content="initial-scale=1, width=device-width" />

or

&viewport { width: device-width; /* not yet adopted by most browsers, but should be in future */ }

Making Scalable Images

  • Shouldn't send full image, unless no other choice, as uses up bandwidth and also time to scale the image to correct size for viewport
  • Shouldn't scale up images unless necessary
/* note figure tag, not img */ figure { overflow: hidden; /*cuts off overflow */ overflow: scroll; /* adds a scrollbar and doesn't cut image */ overflow: auto; /* adds a scrollbar iff necessary */ }
